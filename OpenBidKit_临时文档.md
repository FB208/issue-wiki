# 易标是什么

易标投标工具箱是一款面向招投标场景的 AI 标书写作工具，目标是让标书编写过程更轻量、更可控。

它不是单纯的“生成一段文字”，而是围绕投标工作流提供多个辅助能力，包括招标文件解析、技术方案生成、商务标辅助、企业知识库、标书查重、废标项检查等。

适合的使用场景：

- 快速整理招标文件要求
- 生成技术标初稿
- 复用企业历史案例和方案素材
- 检查标书中的重复内容
- 辅助排查可能导致废标的响应问题
- 让标书编写过程更容易修改和沉淀

当前项目仍在持续开发中，建议把它作为“AI 辅助写标书工具”使用，最终内容仍需要人工审核、补充和定稿。

# 基础使用流程

使用易标编写标书时，可以按下面的流程进行：

1. 准备招标文件
   将招标文件、技术要求、评分办法、格式要求等资料整理好。

2. 解析招标内容
   使用文档解析能力提取项目背景、服务范围、技术要求、响应条款等信息。

3. 生成标书目录
   根据招标要求生成初步目录，再人工调整章节顺序和章节名称。

4. 生成正文内容
   按章节逐步生成技术方案、实施计划、服务保障、项目团队、质量控制等内容。

5. 补充企业资料
   从企业知识库中引用公司介绍、案例、资质、人员经验、服务能力等素材。

6. 检查和修改
   对生成内容进行人工审阅，并使用查重、废标项检查等能力辅助排查风险。

7. 导出和排版
   将最终内容整理为 Word 或其他投标文件格式，完成排版、签章和提交。

# AI 配置说明

易标支持 OpenAI-like API 形式的模型服务，可以根据团队情况接入不同模型。

常见配置方向：

- 在线大模型：适合质量优先、稳定生成正文内容。
- 本地模型：适合对数据安全要求较高，或希望降低调用成本。
- 生图模型：可用于辅助生成方案示意图、流程图等视觉内容。
- 文档解析服务：可用于处理复杂 PDF、扫描件或较长招标文件。

建议配置时优先关注三点：

- 模型是否稳定输出长文本
- 是否支持较长上下文
- 成本是否适合频繁生成和反复修改

AI 生成内容只能作为初稿，不建议直接提交。涉及承诺、资质、价格、工期、响应条款的内容，必须由业务人员或投标负责人复核。

# 企业知识库怎么用

企业知识库用于沉淀可复用的投标素材，让 AI 生成内容更贴近企业实际情况。

适合放入知识库的内容：

- 公司介绍
- 企业资质
- 项目案例
- 产品或服务说明
- 技术方案模板
- 售后服务方案
- 项目团队介绍
- 常用承诺和响应说明

使用建议：

- 按主题分类整理，不要把所有资料放在一个大文档里。
- 尽量使用清晰标题，例如“公司简介”“数据安全服务案例”“售后服务承诺”。
- 保留真实可用的信息，删除过期资质、旧联系方式和不再适用的案例。
- 生成标书后仍要人工检查，避免 AI 引用不适合当前项目的内容。

知识库的价值不在于一次性导入很多资料，而在于持续沉淀高质量、可复用的企业素材。

# 标书查重与废标项检查

标书查重和废标项检查是投标文件提交前的重要辅助步骤。

标书查重主要关注：

- 大段内容重复
- 不同章节表达雷同
- 套用模板痕迹明显
- 同一项目中多处描述不一致

废标项检查主要关注：

- 是否遗漏必要响应
- 是否偏离招标文件要求
- 是否出现不该承诺的内容
- 是否存在格式、签章、附件等风险
- 是否有工期、付款、服务范围等关键信息冲突

这些检查不能替代人工审标，但可以作为提交前的辅助清单，帮助减少低级错误。

建议在最终导出前至少检查一次，尤其是技术响应、商务条款、资格文件和评分办法相关章节。

# 适合优先共创的功能

如果希望逐步完善易标，可以优先关注这些方向：

1. 招标文件解析优化
   提升复杂 PDF、扫描件、多附件招标文件的解析稳定性。

2. 标书目录生成
   根据评分办法、技术要求和响应格式生成更合理的目录结构。

3. 正文生成质量
   优化长文本生成、章节衔接、语气一致性和内容可控性。

4. 企业知识库
   改善资料管理、引用方式和素材复用体验。

5. 查重与风险检查
   增强重复内容识别、废标项提示和响应完整性检查。

6. 导出格式管理
   优化 Word 导出、样式模板、图表和目录排版。

当前阶段可以先围绕真实投标流程提出小需求，逐步把高频、痛点明确的能力打磨稳定。
