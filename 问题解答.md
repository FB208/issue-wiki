已阅读 `方案.md`。下面是我认为需要先补全/确认的问题，按优先级整理。

**核心定位**
1. 这个平台是只服务一个 GitHub 开源项目，还是支持多个项目？
    只服务于一个github项目，但可通过.env配置指定不同的github项目
2. 是否需要和 GitHub 仓库打通，例如展示 issue、PR、版本、贡献者，还是只是“面向 GitHub 项目的独立平台”？
    需要打通，比如github登录、自动同步issue等，但第一版先不做
3. 平台是否公开给所有人访问，还是仅特定项目社区使用？
    本项目也是开源的

**角色权限**
1. 管理员如何创建？是否只有初始化管理员，还是后台可新增管理员？
    通过.env配置管理员邮箱和密码
2. 普通用户和游客的权限边界是否如下：游客可浏览、点赞、赞助；用户可提需求、评论、共创、查看自己的任务和赞助记录？
    是的
3. 是否需要封禁用户、删除评论、隐藏任务、审核评论等管理能力？
    需要封禁用户、删除评论、隐藏任务。评论不需要审核，也不需要确认状态；管理员可以回复或删除评论。

**任务管理**
1. 用户“提需求”创建的任务字段是否和管理员录入任务一致？例如是否允许填写启动资金、顺序、描述？
    展示上是一致的，但从管理上要区分，不支持填写启动资金、顺序，可以写详细描述。
2. “待审核”任务如何进入后续状态？是“待启动”还是由管理员选择？
    不再保留独立审核操作，由管理员直接把任务状态改为待启动、进行中或已完成。
3. “启动资金”含义是什么？是达到该金额后任务自动变成“待启动/进行中”，还是仅展示目标金额？
    这是给用户看的募集赞助的目标金额，不会自动改变任何状态
4. “已赞助金额”达到启动资金后，是否需要自动改变任务状态？
    这是给用户看的，不会自动改变任何状态
5. “共创人数”如何计算？按评论用户数、赞助用户数，还是评论+赞助去重人数？
    仅评论的人数（任务的共创和评论是一个意思）
6. 任务完成后是否还能继续赞助、评论、共创？
    可以继续赞助，但不能共创了
7. 任务排序中的“顺序”是否必须唯一？管理员是否可以拖拽调整顺序？
    必须唯一、拖拽排序

**赞助与支付**
1. 游客赞助时是否需要填写昵称、邮箱、留言，还是完全匿名？
    已登录用户自动记录登录信息，未登录用户统一记录为游客。
2. 已登录用户的赞助记录是否对本人可见？是否需要后台订单管理？
    本人可见，其他用户先不考虑，后续会加赞助榜单，所以要在数据库里记录。初版先不做展示。
3. 是否需要展示公开赞助榜、赞助人列表、赞助留言？
    初版先不做
4. 支付订单有哪些状态？例如待支付、已支付、支付失败、已关闭、已退款。
    你自己读支付接口文档
5. 是否支持退款？如果任务被拒绝、取消或长期未启动，赞助金额如何处理？
    不支持退款
6. 爱发电 webhook 如何处理重复通知、未备注功能 ID、订单无法归属等异常？
    你自己读文档
7. 最低赞助金额、最高赞助金额、金额精度是否有限制？
    最低10元

**文档功能**
1. 左侧菜单中文档和文件夹是否需要自定义排序？
    支持
2. 文档是否需要草稿/发布状态，还是管理员保存后立即公开？
    保存后立即公开
3. 文档是否需要版本历史、回滚、删除恢复？
    不需要
4. 游客可以点赞文档，但游客点赞如何防刷？按 IP、浏览器指纹，还是不严格限制？
    不严格限制
5. 用户评论文档是否需要审核？是否支持删除、置顶、回复？
    评论不需要审核，也不需要确认状态；管理员可以回复或删除评论。

**Markdown 与附件**
1. Markdown 图片和附件上传后存储在哪里？本地磁盘、对象存储，还是 Docker 挂载目录？
    我用rustfs搭建了图床，存到图创立，图床配置在.env
2. 上传附件只在任务共创中支持，还是文档编辑器也支持？
    支持，封装统一的markdown编辑器
3. 附件允许哪些类型？是否只限制 10MB，还是还要限制扩展名和 MIME 类型？
    只支持图片、pdf、word、excel、md几种格式
4. Markdown 是否需要支持代码高亮、目录锚点、任务列表、表格、Mermaid 图表？
    找个轻量级的markdown编辑器工具就行，不必自己实现

**注册登录**
1. 注册时“邮箱/手机号”是二选一，还是必须都支持且任选？
    二选一
2. 手机号注册是否必须走阿里云短信验证码？邮箱注册是否也需要验证码？
    手机号必须走阿里云短信验证码，邮箱注册也需要验证码
3. Gmail 邮箱是指用 Gmail SMTP 发邮件，还是仅允许 Gmail 邮箱注册？
    用我的gmail smtp发验证码，支持所有邮箱注册
4. 忘记密码验证码是否邮箱和短信都支持？
    注册用的什么，找回就用什么
5. 是否需要登录态过期、记住登录、多端登录控制？
    登录状态长期不过期
**后台管理**
1. `/admin` 是否和前台同一个 React 应用，还是独立后台页面？
    同一个
2. 管理员后台需要哪些模块？任务管理、文档管理、用户管理、订单管理、评论管理、系统配置是否都需要？
    你根据需求自己思考，需求提到就加，有用就加，没用不加
3. 后台操作是否需要审计日志，例如谁审核了任务、谁修改了文档？
    不需要

**部署与运维**
1. Docker 是单容器包含前后端，还是 MySQL、Redis 也由 docker-compose 一起部署？
    只包括前后端
2. Linux 服务器部署方式是什么？SSH 拉取镜像、直接构建，还是推送到镜像仓库？
    参考deploy-backend.yml，我给你放到根目录了，不是一模一样，这是另一个项目的actions，你理解它做的工作，然后实现本项目的actions
3. 数据库迁移工具是否需要指定？例如 Alembic。
    Alembic
4. 是否需要备份策略，尤其是 MySQL 数据和上传附件？
    不需要
5. 域名、HTTPS、反向代理是否由项目内处理，还是服务器已有 Nginx？
    不在项目内处理
